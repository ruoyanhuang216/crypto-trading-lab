## Work completed

### P-ML10 — Risk Overlay (F17) ✅

Created `ml/risk/overlay.py` with `RiskOverlay` class implementing two post-prediction
risk controls:

1. **Drawdown brake:** when 30-bar rolling equity DD < −20%, halve all positions
2. **Bull cap:** cap bull-regime long positions at 0.5

**Results (5-fold walk-forward, 6yr BTC/USDT daily):**

| Strategy | Sharpe | Return | MaxDD |
|---|---|---|---|
| P-ML9 binary (baseline) | +1.234 | +1815% | −77.3% |
| DD brake only | +1.334 | +2319% | −68.4% |
| Bull cap only | +1.146 | +1239% | −77.3% |
| Combined (DD+bull) | +1.273 | +1729% | −68.4% |
| P-ML9 scaled (champion) | **+1.583** | +759% | **−33.6%** |
| Combined on scaled | +1.518 | +646% | −33.2% |

**Key insight:** DD brake helps binary signals (Sharpe +1.234 → +1.334) but is redundant
on scaled signals. P-ML9 scaled positioning remains the champion. Bull cap hurts because
it cannot distinguish early vs late bull.

### P-ML11 — HMM Regime Classifier (F18) ✅ — Hypothesis H4 REJECTED

Created `ml/regime/hmm.py` with `HMMRegimeClassifier` using a 4-state Gaussian HMM
to detect late-bull overextension.

**HMM observations:** ret_20, atr_pct, mom_zscore_20, ret_5_minus_20.

**Results:**

| Strategy | Sharpe | Return | MaxDD |
|---|---|---|---|
| Baseline (P-ML7 binary) | **+1.234** | +1815% | −77.3% |
| Exp-A (HMM one-hot features) | +1.074 | +1043% | −77.2% |
| Exp-B (gate=0.5, block late-bull) | +1.055 | +884% | −76.8% |

**Fold 2 bull IC:** −0.128 → −0.132 (no improvement).

**Why H4 failed:**
1. HMM observation features overlap with existing FEATURES_V2 — one-hot is lossy discretization
2. Gating late-bull is too blunt (blocks entire overextension, not just crash bars)
3. Fold 2 is a timing problem, not a classification problem

**Files created:**

- `notebooks/p_ml11_hmm_regime.ipynb`
